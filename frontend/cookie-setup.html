LinkedIn Session Setup

Copy your li_at session cookie from your browser's DevTools. No password needed on the server.

⚠️ The console script method does not work — li_at is an httpOnly cookie, which means JavaScript cannot read it. You must copy it manually from DevTools → Application → Cookies.
1

Open LinkedIn and confirm you're logged in

Visit linkedin.com in Chrome, Edge, or Firefox. Make sure your feed is visible.

2

Open DevTools

Windows / Linux: F12  |  Mac: Cmd+Option+I

3

Go to Application → Storage → Cookies

Click the Application tab at the top of DevTools.
In the left sidebar: Storage → Cookies → https://www.linkedin.com

💡 Don't see "Application"? Click the ≫ overflow arrow to find hidden tabs.
4

Find the li_at row and copy its value

Scroll or use Ctrl+F/Cmd+F to find li_at.
Click the row → at the bottom panel you'll see the full value. Select all and copy it.
It's a long string (~400 chars) that starts with AQE.

🔒 This value is your session token — treat it like a password. It expires when you log out of LinkedIn.
5

Paste below and save

Paste the copied value into the field below and click Save Session.

Required. ~300–600 characters. Found in DevTools → Application → Cookies → linkedin.com
